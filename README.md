# AI狼人杀远程联机版

这是基于你提供的 `AI狼人杀唯一最终版.html` 改出来的可部署版本，保留了原本的核心玩法与配置项，并新增：

- 创建 / 加入房间
- 公开房间
- 带 6 位数字密码的私人房间
- 房主固定为 1 号玩家
- 多人远程联机的人类席位加入
- AI 请求服务器侧转发
- 人类玩家可点击 `📣 加速` 给所有未完成 AI 请求发送竞速副本
- 最快返回的副本会立即胜出，并中断同提示词其他副本

## 保留的原有功能

- 4 个游戏模式选择
- 管理员模式开关
- 自定义发牌
- 日志
- 性格设置
- API 自动模式
- AI 限制字数
- AI 强化
- 发言顺序设置
- 看牌 / 管理员击杀复活 / 夜间与白天流程 / 遗言 / 猎人 / 白痴等原逻辑

## 运行

```bash
npm install
npm start
```

默认端口：`3000`

## Railway 部署

1. 把整个项目上传到 GitHub 仓库
2. 在 Railway 里选择 **Deploy from GitHub repo**
3. 选中这个仓库
4. Railway 会自动执行：
   - `npm install`
   - `npm start`
5. 部署完成后即可得到在线链接

## 使用方式

### 房主

1. 进入首页后创建公开房间或私人房间
2. 如果是私人房间，会生成你设置的 6 位数字密码
3. 房主默认固定为 1 号位
4. 在原来的配置界面里把需要远程加入的席位切换成“人类”
5. 其他玩家进入房间后，选择对应的人类席位加入
6. 房主配置好模式 / API / 发牌 / 规则后点击开始游戏

### 其他玩家

1. 通过房间号加入
2. 私人房间需要输入 6 位密码
3. 在大厅里点击可用的人类席位加入
4. 游戏开始后，在自己的设备上完成身份确认、夜间行动、发言、投票、遗言等

## 喇叭加速说明

只要房间里存在**仍在等待中的 AI 请求**，人类玩家就可以点击 `📣 加速`：

- 服务器会给当前还没返回的每个 AI 请求再发一个副本
- 多个副本并行竞速
- 谁先返回就采用谁
- 同一提示词的其他副本会被立即中断
- 连续点击会继续追加副本

## 重要说明

1. **房主页面需要保持打开**。当前版本的游戏流程推进仍由房主浏览器作为裁判主控来驱动。
2. AI 接口调用会经过当前 Railway 服务端进行转发与竞速处理。
3. API 配置由房主在浏览器里输入，并同步到房间所在的服务进程内存中；服务重启后需要重新填写。
4. 房间与对局状态也是内存态，服务重启后不会保留。

## 文件结构

- `server.js`：Express + Socket.IO 服务端
- `public/index.html`：页面入口
- `public/app.jsx`：前端主逻辑（基于原 HTML 脚本改造）
- `package.json`：依赖与启动脚本
